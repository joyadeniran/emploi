## Design system

Tokens in `web/app/globals.css` under `@theme` (Tailwind v4):

- Brand: `--color-brand #5b4ffd`, gradient `#8b48fd → #435afe`, ink `#04114d`, muted `#4b5476`, surface `#f7f6fc`, line `#ecebf6`; semantic good/warn/amber/info pairs with `-soft` backgrounds.
- Type: Plus Jakarta Sans (next/font, `--font-jakarta`), extrabold headings, tight tracking.
- Shape: cards `rounded-2xl border border-line bg-white shadow-card`; pills `rounded-full`; primary buttons gradient `from-brand-violet to-brand-indigo shadow-pop hover:-translate-y-0.5`.
- Motion: `.rise-in` entrance; rings animate via stroke-dashoffset; everything respects `prefers-reduced-motion`.
- The marketing site (`landing/`) uses the same palette with glassmorphism; the app is flat/clean (mockup-accurate).

## Components (web/components/)

`AppShell` (drawer state), `Sidebar` (nav + Upgrade card + plan meter), `Topbar` (search, bell, account menu with sign-out), `Logo`/`LogoMark`, `ProgressRing`/`FitRing`, `CareerTwinBot` (SVG illustration), `ApplyButton` (optimistic POST), `PagePlaceholder`.

## Data flow

- Server components call `lib/api.ts apiFetch()` (server-only; injects `X-API-Key` + `X-User-Id` from the NextAuth session).
- Client components call same-origin route handlers (`app/api/*`) which proxy to the API — the shared secret never reaches the browser.
- **Fallback rule:** when the API is unreachable, pages show demo data with an explicit "sample data" banner — never silently.
- No client state library yet. Add React Query only when cross-page cache invalidation actually hurts.

## State & UX conventions

- Optimistic updates with revert-on-failure (see Applications status select, ApplyButton).
- Controlled inputs; `aria-pressed`/`aria-expanded`/`aria-label` on interactive elements; `role="menu"` for menus.
- Currency: Naira with explicit ₦; dates human-readable.

## Acceptance criteria

- `npm run build` + `npm run lint` clean.
- No horizontal overflow at 375px on any page (DOM scan, not eyeballing).
- Sign-in → dashboard → apply → applications → status change → reload persists (manual E2E, browser).
- API down ⇒ every live page still renders with its fallback banner.

## Edge cases

- Hydration mismatch from browser extensions → `suppressHydrationWarning` on `<body>` only.
- User with no `image` → initials avatar. No name → "there"/"Account" fallbacks.
