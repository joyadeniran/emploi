# CLAUDE.md — Emploi

Emploi is an AI job-application agent: a Streamlit chat app where users drop a CV (PDF), load job listings (PDF/CSV/XLSX), and get verified, ranked, tailored applications generated by Google Gemini. Target users: remote job seekers in Africa/Asia applying to jobs from curated sheets and informal sources. Differentiator: employer trust verification (scam protection).

Read this whole file before changing anything. The invariants below are load-bearing; violating them breaks tests or the product promise.

## File map

| File | Role |
|---|---|
| `app.py` | Streamlit chat UI ONLY. Thin dispatch layer. No business logic here. |
| `core.py` | ALL logic: prompts, intent routing, CV/job extraction, generation, matching, exports. UI-free, importable without Streamlit. |
| `verify.py` | Employer trust engine. Deterministic scoring. All I/O injectable. |
| `skills/*.md` | Prompt modules injected into Gemini calls at runtime. Editing them changes behavior with zero code changes. |
| `test_e2e.py` | Core suite (fake Gemini model, no network, no API key). |
| `test_verify.py` | Verification suite (fake DNS/HTTP/LLM, no network). |
| `CHANGELOG.md` | Append an entry for EVERY shipped change. |
| `render.yaml`, `Dockerfile`, `DEPLOY.md` | Deployment (Render free tier / Cloud Run). |

## Load-bearing invariants

1. **Logic lives in `core.py`/`verify.py`, never `app.py`.** If you're writing an `if` that isn't UI dispatch inside `app.py`, you're in the wrong file. Everything in core/verify must run without Streamlit installed.

2. **Every model object is duck-typed.** Functions take a `model` with `.generate_content(prompt) -> obj.text`. Tests pass `FakeModel`. Never call `genai` directly inside core/verify functions; the model is always passed in.

3. **Trust scores are computed in code, never by an LLM.** `verify.compute_trust()` maps named signals to point deltas. Gemini's only verification role is `check_site_content()` returning consistent/inconsistent/None. Do not add "ask the model for a trust score" logic. Red flags cap the score at 35; no contact caps at 40. Failed network probes produce absent/None signals ("unverified"), never fabricated ones.

4. **The fit-score contract:** generated applications must end the evaluation with a line matching `Fit Score: NN/100` — `parse_fit_score()` (regex `FIT_RE`) depends on it. If you change the output format in `build_prompt`, update `FIT_RE`, `build_review_prompt`'s structure rule, and the tests together.

5. **JSON parsing from Gemini is defensive.** Model output may include ```json fences and prose. Always go through `parse_profile_json` / `parse_json_array` (which use `_extract_json`); never `json.loads(resp.text)` directly. Parsers return `{}`/`[]` on garbage, never raise.

6. **Intent routing (`detect_intent`) is ordered and slash-tolerant.** Order: pdf upload → sheet upload → `apply` → `interview` → `verify` → `batch` → `tracker` → `match` → long-text/JD-hints → chat fallback. A leading `/` is stripped first. Command regexes only match short texts (length guards) so pasted JDs containing the word "apply" don't misroute. New intents need: regex here, handler branch in `app.py`, tests for bare AND `/`-prefixed forms.

7. **Skills are the prompt system.** `load_skill(name)` reads `skills/<name>.md` (cached in `_skill_cache`). Generation and review prompts inject `writing_style` + `evaluation`; matching injects `evaluation`; interview prep injects `interview_prep`. Behavior tuning belongs in the skill files, not hardcoded prompt strings. Tests assert marker phrases from skills appear in built prompts ("NO em-dashes", "weight 30%") — if you rewrite a skill, keep or update those markers.

8. **Never fabricate candidate experience.** Every prompt carries a ground-truth constraint; the writing-style skill defines the "interview backtrack test". Do not weaken these when editing prompts. Stretchy CV bullets are marked "(stretch — verify)".

9. **fpdf2 gotcha (was a real bug):** `multi_cell` must pass `new_x="LMARGIN", new_y="NEXT"`, and empty lines go through `pdf.ln(6)` — otherwise fpdf2 raises "Not enough horizontal space". Don't "simplify" `make_pdf`.

10. **API key is server-side.** Resolved from `GEMINI_API_KEY` env var or `st.secrets`; the sidebar input is a dev-mode fallback only. Never log, echo, or commit a key. `.streamlit/secrets.toml` is gitignored.

11. **Session-state keys** (`profile`, `applications`, `jobs`, `matches`, `messages`, `verify_cache`, `last_jd`, `last_company`) are initialized via `st.session_state.setdefault` at the top of `app.py`. Add new ones there, nowhere else. All data is per-session and in-memory by design (v1); don't add persistence casually — it's a roadmap item with its own design.

12. **Jobs are dicts with flexible schemas.** Extracted jobs have `company/title/description/contact`; sheet imports have arbitrary columns and go through `guess_columns()` (longest-average-text column = description; company matched by name or exact terse headers like "co"/"org"). Any code consuming jobs must handle both shapes.

13. **Verification results are cached per domain** (`cache` dict param). The cache test asserts network probes run exactly once per domain — preserve cacheability when touching `verify_employer`.

## Testing discipline

- Every behavior change ships with a test in the same commit. Tests are plain scripts with `check(label, cond)` lines — follow that style, no pytest.
- Tests must stay offline: fake models, injected `dns_fn`/`mx_fn`/`fetch_fn`. If your test needs real network or a real API key, it's wrong.
- Bug fixes get a regression test reproducing the original report (see the slash-command tests for the pattern).
- Both suites green before claiming done. Then append a CHANGELOG entry.

## Product guardrails (don't regress these)

- Low-trust employers trigger a warning on `apply` ("never pay a fee or share bank/ID details") — keep it.
- Scanned/image-only PDFs are detected (< 50 chars extracted) and get a helpful message, not a crash.
- Every user-facing failure path answers in chat via `say("assistant", ...)`; exceptions must never reach the Streamlit error screen.
- Reviewer pass doubles API calls; anything adding calls per job must be mentioned in the UI (see batch caption).

## Deploy notes

- Streamlit cannot run on Vercel/Netlify (serverless) or Hostinger shared (PHP-only). Render free tier (`render.yaml`, sleeps after ~15 min idle) or Cloud Run (`Dockerfile`, `gcloud run deploy emploi --source .`).
- Gemini model names change; the selector lives in `app.py` sidebar. Default `gemini-2.5-flash`.
